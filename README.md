# HyperDA-Leaf

## Overview
HyperDA-Leaf is a DSSAT-based data assimilation framework for jointly updating daily LAI and FAPAR using UAV observations and canopy structural dynamics. The framework is implemented by modifying the DSSAT maize canopy growth module (`MZ_GROSUB.for`) and linking DSSAT with an external Python assimilation script.

## Workflow
During daily DSSAT simulation, canopy structural variables are first exported from DSSAT, including LAI, PLA, SENLA, PLAG, and PLAS. These variables are then passed to the Python script `assimilate_lai_fapar.py`, which performs LAI–FAPAR assimilation and writes the corrected LAI and FAPAR values to an external text file. DSSAT then reads the corrected values back and uses them in the subsequent physiological calculations.

## Software Environment
- Windows
- DSSAT v4.8.2
- Python 3.11
- Visual Studio (for recompiling DSSAT source code)

## DSSAT Source Code Modification
The framework is built on the DSSAT Cropping System Model (CSM) source code. The maize canopy growth module `MZ_GROSUB.for` was modified to:

1. export daily leaf structural variables;
2. call an external Python assimilation script;
3. re-import corrected LAI and FAPAR into DSSAT;
4. replace the original LAI-based intercepted PAR calculation with assimilated FAPAR;
5. record diagnostic outputs such as daily FAPAR and leaf stress factors.

## External Files Used in DSSAT–Python Interaction
The modified DSSAT code uses the following external files:

- `LAIandLeafDynamics.TXT`  
  Daily exported canopy variables, including LAI, PLA, SENLA, PLAG, and PLAS.

- `LAI_FAPAR_Assimilated.TXT`  
  Corrected LAI and FAPAR generated by the Python assimilation script.

- `FAPARrecord.TXT`  
  Daily assimilated FAPAR record for diagnostic purposes.

- `LeafStress.TXT`  
  Daily stress factors recorded during simulation.

## How to Use

### Step 1. Obtain DSSAT CSM Source Code
Download or clone the official DSSAT CSM source code from the DSSAT GitHub repository.

### Step 2. Modify DSSAT Source Code
Open the generated `DSSAT-CSM.sln` solution in Visual Studio and modify `MZ_GROSUB.for` according to the documentation provided in `DSSAT_Source_Code_Compilation_and_Modifications.pdf`.

### Step 3. Recompile DSSAT
Rebuild the DSSAT-CSM project in Visual Studio to generate the updated executable (`dscsm048.exe`).

### Step 4. Replace DSSAT Executable
Back up the original DSSAT executable in your DSSAT installation folder and replace it with the recompiled version.

### Step 5. Prepare Assimilation Inputs
Place the required observation and boundary files in the expected working directory, including:
- `LAI_observations.xlsx`
- `FAPAR_observations.xlsx`
- `lai_hybrids_bounds22.xlsx`
- `fapar_hybrids_bounds22.xlsx`

### Step 6. Place the Python Assimilation Script
Make sure `assimilate_lai_fapar.py` is located in the path expected by the modified DSSAT code, or revise the file path in `MZ_GROSUB.for` accordingly.

### Step 7. Run DSSAT
Run DSSAT normally. During simulation, DSSAT will automatically:
1. export daily canopy structural variables;
2. call `assimilate_lai_fapar.py`;
3. read corrected LAI and FAPAR back into the model;
4. continue the simulation using the assimilated state variables.

## Notes
